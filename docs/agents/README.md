# Agent Documentation

This directory contains workflow documents written primarily for LLM coding
agents and automation.

Use this directory for:

- step-by-step agent playbooks;
- autonomous editing, tracing, and verification workflows;
- guidance that helps an agent decide which committed docs, skills, scripts,
  and validation targets to open next.

Do not put product requirements, architecture contracts, deployment rules, or
test ledgers here. Keep those in `docs/product/`, `docs/architecture/`,
`docs/operations/`, and `docs/testing/`.

Start with:

- [agent-playbook.md](agent-playbook.md)
- [local-agent-workspace.md](local-agent-workspace.md)
